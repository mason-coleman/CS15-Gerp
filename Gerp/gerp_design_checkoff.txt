4. Explain what a collision is versus having multiple lines associated with 
   a particular word. Write (and/or draw) out an example of when a 
   collision occurs. Write (and/or draw) out an example of when multiple 
   lines associated with a single word. 

Answer:
A collision occurs when after two different words are hashed, they end up 
at the same line index. This can be dealt with in two ways: chaining and 
open addressing. We will utilize chaining for this project, where we will
add a hash node to a linked list for each collision.

Multiple lines associating with the same word occurs when a word appears
in multiple different lines of text. In our hashnodes, we will store an
array of integers to store each line number.

8. How will you handle case-sensitive search?
   What is the time complexity (should be close to O(1))?

Answer: For case-sensitive search, we strip the query with stripNonAlphaNum and 
look it up in table, where words were stored as originals (ex."COMP" and "comp" 
were inserted as separate keys). The time complexity is O(1). 

9. How will you handle case-insensitive search?
   What is the time complexity (should be close to O(1))?

Answer: For case-insensitive search, we strip the query with stripNonAlphaNum 
and then lowercase every character. We then look up this lowercased key. At 
build time, every word was also lowercased before insertion into this table
(ex. "COMP", "Comp", and "comp" all map to the single key "comp"). The time 
complexity is essentially O(1). 

10. How are you handling duplicate words within a line? Consider these two example

      (i) the cat and the dog 
      (ii) the cat and THE dog 

   If a user does a case sensitive query on line (i) it should report 'the' only 
   once by showing the line only once in the output. How will you do this? 

   If a user does a case insensitive query on line (ii) it should report 'the' 
   only once by showing the line only once in the output. How will you do this? 

Answer:
In our HashNodes we are storing vectors of each line number. Since we have 
seperate tables for sensitve and insensitive words, we just ensure to only
print the same line number once. We can do this by making sure that every
new line index we add in the building phase is not the same as the one
currently at the back of the array.
